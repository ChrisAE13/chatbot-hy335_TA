Ενώ έχω ορίσει όλα τα interface ips και port ips σωστά στους routers (όπως μπορώ να διαπιστώσω και από τα αποτελέσματα της εντολής show interface), και ενώ τα γειτονικά interfaces έχοyν σωστά ips (βλ. εντολή show ip ospf neighboor) τα neighbor ids είναι λάθος. Π.χ. μεταξύ LOND & PARI εκτελώντας  show ip ospf neighboor παρατηρώ πως σαν neighbor id (1η στήλη) το PARI έχει την ip του Interface dns_87 του LOND router αντί για την lo ip, παρόλο που ως interface (4η στήλη) έχει σωστή την ip (Χ.0.4.1). Ομοίως  neighbor id (1η στήλη) το LOND έχει την ip του PARI ssh interface, παρόλο που ως interface (4η στήλη) έχει σωστή την ip (Χ.0.4.2). Τι πάει λάθος;;;

το ospf (κατά την έναρξη του) ορίζει σαν router-id την υψηλότερη IP από τα loopback interface (στην περίπτωση του project αφού έχετε μόνο ένα loopback interface το router-id θα πάρει την IP αυτή), αν όμως το loopback interface δεν έχει IP το ospf θα θέσει σαν router-id την υψηλότερη από οποιοδήποτε interface του router (στην περίπτωσή π.χ. την IP ssh interface ή την IP dns interface). Άρα το πρόβλημα δημιουργήθηκε επειδή δεν είχες αναθέσει τις IPs στα loopback interface (όπως αναφέρει στην εκφώνηση και στο φροντιστήριο) πριν ξεκινήσεις να κάνεις configure το ospf με αποτέλεσμα να αναθέσει σαν router-id την υψηλότερη από οποιοδήποτε interface του router.
Το router-id θα πρέπει να είναι μοναδικό για κάθε router στο δίκτυο σας, διαφορετικά αν έχεις routers με το ίδιο router-id θα έχεις reachability προβλήματα (π.χ. αν 2 γειτονικοί routers έχουν το ίδιο id ο ένας δεν θα βλέπει τον άλλο σαν γείτονα στο ospf). Για να δεις το router-id ενούς router εκτέλεσε την εντολή " show ip ospf". Σε περίπτωση που δεν έχεις 2 router στο AS σου με το ίδιο router-id κατά πάσα πιθανότητα δεν θα αντιμετωπίσεις κάποιο πρόβλημα. Αν όμως έχεις 2 router στο AS σου με το ίδιο router-id θα πρέπει να αλλάξεις τα router-ids τους. Για να το κάνεις αυτό θα πρέπει να κάνεις restart το τρέχον instance του ospf σε αυτούς τους routers. Πριν κάνεις restart σημείωσε κάπου τα βάρη που έχεις ορίσει γιατί μπορεί να χαθούν. Για κάνεις restart θα πρέπει να απενεργοποιήσεις το ospf (μόνο στους router που έχουν το ίδιο id): θα μπειs στο configuration mode (conf t) και θα γράψεις "no router ospf" και έπειτα θα κάνεις exit.
